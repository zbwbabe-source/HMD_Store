# Store Dashboard Logic

## 1. 목적

이 문서는 `Store_Dashboard`의 매장 월별 Sales / YoY 화면이 어떤 데이터 소스를 사용하고,
기준월에 따라 실제 실적과 예상 매출을 어떻게 합치는지 설명합니다.

특히 아래 항목을 빠르게 확인할 수 있도록 정리했습니다.

- 실적 기준월(`actualPeriod`)의 의미
- SQL actual과 Excel forecast의 병합 규칙
- TW 환율 적용 규칙
- 화면의 지역 / 브랜드 / 카드 / 표 컬럼 동작 방식

## 2. 기준 파일

- 기준월 설정: `data/store-view-settings.json`
- SQL + Excel 병합 스크립트: `scripts/export_store_monthly_sales_sql.py`
- Snowflake actual 조회: `scripts/fetch_snowflake_actuals.mjs`
- Forecast 원본: `Store_Rawdata.xlsx`
- TW 환율 원본: `TW_Exchange Rate.csv`
- 화면 컴포넌트: `components/dashboard-shell.tsx`

## 3. 실적 기준월(`actualPeriod`)

`data/store-view-settings.json`의 `actualPeriod`가 이 화면의 기준월입니다.

예:

- `2026-02`이면 2026년 2월까지 actual로 간주
- `2026-03`이면 2026년 3월까지 actual로 간주

스크립트는 이 값을 `actual_year`, `actual_month`로 분리해서 사용합니다.

## 4. 데이터 소스 구조

이 화면은 데이터를 2개 소스에서 가져옵니다.

### 4.1 SQL actual

Snowflake 테이블 `SAP_FNF.DW_HMD_SALE_D`에서 실제 판매 금액을 읽습니다.

주요 규칙:

- 매장별 / 연도별 / 월별로 집계
- 금액은 `SUM(ACT_SALE_AMT) / 1000`
- 즉 SQL 결과도 `1k HKD` 기준으로 맞춰서 가져옴

### 4.2 Excel baseline / forecast

`Store_Rawdata.xlsx`를 기준 데이터로 사용합니다.

이 파일은 월별 값의 기본 골격 역할을 하며,
기준월 이후 당해연도 forecast는 이 파일 값을 그대로 사용합니다.

## 5. SQL과 Excel 병합 규칙

핵심 규칙은 아래 한 줄로 정리할 수 있습니다.

- 기준연도보다 이전 연도는 전부 SQL
- 기준연도에서 기준월까지는 SQL
- 기준연도에서 기준월 이후는 Excel forecast

예를 들어 `actualPeriod = 2026-02`이면:

- 2024년 전체: SQL
- 2025년 전체: SQL
- 2026년 1~2월: SQL
- 2026년 3~12월: Excel

예를 들어 `actualPeriod = 2026-03`이면:

- 2024년 전체: SQL
- 2025년 전체: SQL
- 2026년 1~3월: SQL
- 2026년 4~12월: Excel

즉, 사용자가 기준월을 뒤로 밀면 당해연도에서 SQL이 덮는 범위도 함께 늘어납니다.

## 6. TW 환율 적용 규칙

TW는 SQL 원천 금액이 TWD 기준이므로 환율 적용이 필요합니다.

다만 모든 월에 같은 방식으로 적용하지는 않습니다.

### 6.1 SQL actual이 적용되는 월

TW의 SQL actual은 월별 환율을 적용해서 HKD로 환산합니다.

중요한 비교 규칙:

- 전년 / 전전년도도 각 연도의 실제 환율을 쓰지 않음
- 선택된 기준연도의 동일 월 환율을 공통 적용

예:

- 2026년 1월 actual 환산: `2601` 환율 사용
- 2025년 1월 비교값 환산: 역시 `2601` 환율 사용
- 2024년 1월 비교값 환산: 역시 `2601` 환율 사용

즉, 환율 차이에 의해 YoY가 왜곡되지 않도록 비교 기준을 맞추는 방식입니다.

### 6.2 당해연도 기준월 이후 forecast 월

TW의 당해연도 forecast 월은 `Store_Rawdata.xlsx` 값이 이미 `1k HKD` 기준입니다.

따라서:

- 기준연도 + 기준월 이후 월은 환율을 다시 적용하지 않음
- Excel 값을 그대로 사용

예:

- `actualPeriod = 2026-02`일 때 TW 2026년 3~12월은 환율 미적용

## 7. 단위

화면의 월별 표와 카드 금액 단위는 `1k HKD`입니다.

정리하면:

- SQL actual: `ACT_SALE_AMT / 1000`
- TW SQL actual: 월별 환율 적용 후 `1k HKD`
- Excel forecast: 이미 `1k HKD`로 간주

## 8. 브랜드 처리

화면에서는 브랜드를 드롭다운으로 선택합니다.

- `MLB`
- `DX`
- `전체`

기본값은 `MLB`입니다.

브랜드 선택은 상단 카드와 하단 월별 표에 함께 적용됩니다.

## 9. 지역 처리

화면 상단 지역 버튼:

- `HKMC`
- `TW`

동작 방식:

- `HKMC`: 홍콩 + 마카오 묶음
- `TW`: 대만 단독

카드의 왼쪽 첫 번째 합계 카드는 지역에 따라 아래처럼 바뀝니다.

- `HKMC` 선택 시: `홍콩마카오 합계`
- `TW` 선택 시: `대만 전체`

## 10. 카드 영역 규칙

상단 카드 영역은 크게 2종류로 나뉩니다.

### 10.1 전체 합계 카드

가장 왼쪽 카드입니다.

표시 항목:

- 실판매출
- YOY
- 전전년비

이 카드는 `최고 매출`, `최고 YOY`를 보여주지 않습니다.

### 10.2 채널 / 국가 하이라이트 카드

오른쪽 카드들은 채널별 또는 지역별 하이라이트를 보여줍니다.

표시 항목:

- 카드 제목
- 선택월 기준 요약(YOY / 전전년비)
- 최고 매출 매장
- 최고 매출의 YOY / 전전년비
- 최고 YOY 매장
- 최고 YOY 값의 YOY / 전전년비

## 11. 카드의 누적 / 당월 토글

카드에는 `누적 / 당월` 토글이 있습니다.

기본값은 `누적`입니다.

### 11.1 누적

- 선택월까지 누적 기준
- 예: 2월 선택 시 1~2월 누적

### 11.2 당월

- 선택한 해당 월만 기준
- 예: 2월 선택 시 2월 단월

이 토글은 왼쪽 합계 카드뿐 아니라 오른쪽 카드들도 함께 바뀝니다.

## 12. 하단 월별 표 규칙

하단 표는 다음 특징을 가집니다.

- 월별 컬럼 표시
- 선택월 바로 오른쪽에 `YTD` 컬럼 추가
- `YoY만 보기` / `매출액 보기` 전환 가능
- 채널 합계 행 확장 가능

행 구조 예시:

- 전체 합계
- 국가 합계
- 브랜드 합계
- 채널 합계
- 개별 매장

## 13. YoY / 전전년비 계산 방식

이 화면에서 표시하는 `YOY 119%`는 증감률 `+19%`가 아니라
`전년 대비 119% 수준`이라는 의미의 index 값입니다.

즉 계산은 아래처럼 합니다.

- `당해 값 / 전년 값 * 100`

전전년비도 같은 방식입니다.

- `당해 값 / 전전년 값 * 100`

## 14. 색상 규칙

카드와 표의 지표 색상 규칙:

- `100% 이상`: 초록
- `100% 미만`: 빨강

이 규칙은 YOY와 전전년비 각각에 개별 적용됩니다.

## 15. 현재 현황과 추세 문구

카드 아래의 `현재 현황과 추세` 문구는 고정 문장이 아니라
현재 선택 상태에 따라 동적으로 생성됩니다.

반영 요소:

- 지역(`HKMC` / `TW`)
- 브랜드(`MLB` / `DX` / `전체`)
- 기준월
- 카드 모드(`누적` / `당월`)
- 실제 계산된 매출 / YOY / 전전년비

즉 사용자가 필터를 바꾸면 추세 문구도 같이 바뀝니다.

## 16. 유지보수 시 가장 많이 헷갈리는 포인트

### 16.1 기준월을 바꿨을 때

기준월을 뒤로 옮기면 당해연도에서 SQL이 차지하는 범위가 늘어납니다.

### 16.2 TW forecast

TW의 당해연도 기준월 이후 forecast는 이미 HKD이므로
환율을 다시 곱하면 안 됩니다.

### 16.3 비교 연도 환율

TW 비교값은 전년의 실제 환율이 아니라
기준연도의 동일 월 환율을 써야 합니다.

### 16.4 YTD

YTD는 선택월의 누적값이며,
선택월이 바뀌면 YTD 기준도 함께 바뀝니다.

## 17. 추천 점검 순서

로직 수정 후에는 아래 순서로 확인하는 것을 권장합니다.

1. `data/store-view-settings.json`의 `actualPeriod` 확인
2. SQL actual이 기준월까지 들어오는지 확인
3. 기준월 이후 당해연도 월이 Excel 값인지 확인
4. TW의 actual 월에 환율이 적용됐는지 확인
5. TW의 forecast 월에 환율이 중복 적용되지 않았는지 확인
6. 브랜드 / 지역 필터에 따라 카드와 표가 함께 바뀌는지 확인
7. `현재 현황과 추세` 문구가 선택 상태에 맞게 바뀌는지 확인

## 18. 한 줄 요약

이 화면은 `기준월까지는 SQL actual`, `그 이후 당해연도는 Excel forecast`,
`TW는 actual만 월별 환율 적용, forecast는 환율 미적용` 구조로 동작합니다.
